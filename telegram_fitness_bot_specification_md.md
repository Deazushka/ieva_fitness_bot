# Telegram Fitness Tracker Bot — Спецификация

---

# 1. Общая концепция

Telegram-бот для отслеживания тренировок с использованием **обычных кнопок (ReplyKeyboard)**.

Основные задачи:

- Начало тренировки
- Выбор категории (день ног, день рук, свободная тренировка и т.д.)
- Добавление упражнений
- Добавление подходов (например: `3 12` или `4 10 50`)
- Возможность добавлять несколько упражнений
- Завершение тренировки с итоговой сводкой
- История тренировок
- Настройки пользователя

Навигация реализована через меню с кнопкой **"🔙 Назад"** на каждом уровне.

---

# 2. Структура навигации

```
MAIN_MENU
   ↓
CATEGORY_SELECT
   ↓
EXERCISE_SELECT
   ↓
EXERCISE_DETAIL
```

Кнопка "🔙 Назад":

| Экран | Возврат |
|------|--------|
| EXERCISE_DETAIL | EXERCISE_SELECT |
| EXERCISE_SELECT | CATEGORY_SELECT |
| CATEGORY_SELECT | MAIN_MENU |
| HISTORY_DETAIL | HISTORY_MENU |
| SETTINGS_MENU | MAIN_MENU |

Правила:

- Не удаляет текущую тренировку
- Не сбрасывает введённые упражнения
- Возвращает только UI уровень

# 3. Экраны интерфейса

## 3.1 MAIN_MENU

Сообщение:

```
🏋️ Добро пожаловать!
Выберите действие:
```

Кнопки (ReplyKeyboard):

```
[🏋️ Начать тренировку]
[📊 История]
[⚙️ Настройки]
[❓ Помощь]
```

---

## 3.2 CATEGORY_SELECT

Сообщение:

```
📋 Выберите категорию тренировки:
```

Кнопки (ReplyKeyboard, пример):

```
[День ног] [День рук]
[День спины] [День груди]
[Свободная тренировка]

[➕ Добавить категорию]
[🗑️ Удалить категорию]

[🔙 Назад]
```

Логика:

- Выбор категории → создаётся новая активная тренировка
- **Важно:** Если у пользователя уже была начата, но не завершена другая тренировка (is_active=True), она безвозвратно удаляется (сбрасывается), и начинается новая.
- Переход к EXERCISE_SELECT

---

## 3.3 EXERCISE_SELECT

Сообщение:

```
💪 Категория: <название>
Выберите упражнение (страница 1/3):
```

Кнопки (InlineKeyboard для списка, внизу ReplyKeyboard для управления):

*Inline:*
```
[Приседания] [Жим ногами]
[Становая тяга] [Выпады]
[⬅️ Назад] [Вперед ➡️]
```

*Reply:*
```
[➕ Добавить упражнение]
[🗑️ Удалить упражнение]

[✅ Завершить тренировку]
[❌ Отменить тренировку]

[🔙 Назад (к категориям)]
```

Логика:

- Выбор упражнения (Inline) → EXERCISE_DETAIL
- Завершить → итоговая сводка → MAIN_MENU
- Отменить → удаление текущей тренировки

---

## 3.4 EXERCISE_DETAIL

Сообщение:

```
💪 <Название упражнения>

Введите подход:

Формат:
повторения [вес]

Пример:
12 50 (12 повторений с весом 50)
15 (15 повторений без веса/со своим весом)
```

Кнопки (динамические пресеты на основе истории пользователя):

```
[10 x 50] [12 x 50]
[15 x 40] [8 x 60]

[🔙 Назад (к упражнениям)]
```

Логика:

- Пользователь вводит данные подхода (или выбирает пресет).
- Бот отвечает: "Записано: 12 повторений, 50 кг." и обновляет пресеты (сохраняет этот вес/повторения для быстрых кнопок в будущем).
- Можно вводить несколько подходов подряд (каждое сообщение = один подход).
- После завершения упражнения пользователь нажимает `[🔙 Назад]` для выбора следующего упражнения.

# 5. База данных (минимальная структура)

## categories
- id
- user_id (NULL для базовых системных, ID пользователя - для кастомных)
- name

## exercises
- id
- category_id
- user_id (NULL для базовых системных, ID пользователя - для кастомных)
- name

## user_presets (Новая: сохранение пресетов)
- id
- user_id
- exercise_id
- reps
- weight
- last_used_at (для сортировки пресетов)

## workouts
- id
- user_id
- category_name
- started_at
- finished_at
- is_active

## workout_exercises
- id
- workout_id
- exercise_name
- reps
- weight
- created_at (время подхода)

# 6. Завершение тренировки

При нажатии:

```
✅ Завершить тренировку
```

Бот:

1. Закрывает тренировку
2. Считает статистику
3. Отправляет summary:

```
✅ Тренировка завершена

Категория: День ног
Упражнений: 3

• Приседания — 3x12 (50 кг)
• Жим ногами — 4x10 (120 кг)
```

Возврат → MAIN_MENU

---

# 7. Инструкция по запуску (высокоуровневая)

1. Создать бота через @BotFather
2. Получить TELEGRAM_BOT_TOKEN
3. Развернуть проект на Railway / Render
4. Добавить переменную окружения TELEGRAM_BOT_TOKEN
5. Проверить запуск через логи

---

# 8. Требования

- Python 3.12
- python-telegram-bot
- SQLite
- ReplyKeyboard (для навигации) + InlineKeyboard (для пагинации списков)
- ConversationHandler (FSM)
