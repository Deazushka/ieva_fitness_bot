# Fitness Tracker Bot - Руководство по развертыванию

## Описание проекта

Telegram-бот для отслеживания тренировок с поддержкой:
- Ведения журнала тренировок
- Истории тренировок
- Настроек пользователя (язык, единицы измерения)
- Мультиязычности (RU/EN)

---

## Требования

| Компонент | Версия |
|-----------|--------|
| Python | 3.12.x |
| Telegram Bot Token | Получить у @BotFather |
| Railway account | https://railway.app |

```

├─────────────────────────────────────┤
│  [• День ног]      [• День рук]     │ → Начать тренировку

│  [• Свободная тренировка]           │
```

**Действия:**
| Кнопка | Callback Data | Действие |

| ➕ Добавить | `add_category` | Запрос названия новой категории |
| 🗑️ Удалить | `delete_category` | Показывает клавиатуру удаления |

### Выбор упражнения

```
┌─────────────────────────────────────┐
│  💪 Выберите упражнение:            │
│  Категория: День ног                │
├─────────────────────────────────────┤
│  [• Приседания]   [• Жим лёжа]      │ → EXERCISE_DETAIL
│  [• Становая]     [• Подтягивания]  │
│  [➕ Добавить]    [🗑️ Удалить]      │
│  [✅ Завершить]   [❌ Отмена]        │
│  [🔙 Назад]                         │ → CATEGORY_SELECT
└─────────────────────────────────────┘
```

**Действия:**
| Кнопка | Callback Data | Действие |
|--------|---------------|----------|
| • Приседания | `ex_Приседания` | Показывает детали упражнения (подходы) |
| ➕ Добавить | `add_exercise` | Запрос названия нового упражнения |
| 🗑️ Удалить | `delete_exercise` | Показывает клавиатуру удаления |
| ✅ Завершить | `finish_workout` | Завершает тренировку, показывает итог |
| ❌ Отмена | `cancel_workout` | Отменяет тренировку (удаляет из БД) |
| 🔙 Назад | `back` | Возврат к выбору категории |

### Детали упражнения (выбор подходов)

```
┌─────────────────────────────────────┐
│  💪 Приседания                      │
│  Выберите подход:                   │
├─────────────────────────────────────┤
│  [📊 3x12]        [📊 5x5]          │ → Быстро добавить
│  [➕ Добавить]    [🗑️ Удалить]      │
│  [🔙 Назад]                         │
│                                     │
│  Или введите: 3 12 [вес]            │
│  Примеры: 3 12, 4 10 50             │
└─────────────────────────────────────┘
```

**Действия:**
| Кнопка | Callback Data | Действие |
|--------|---------------|----------|
| 📊 3x12 | `ex_set_3x12` | Добавляет упражнение в текущую тренировку |
| 🔙 Назад | `back` | Возврат к выбору упражнения |

**Ввод текста:**
```
Формат: "подходы повторения [вес]"
Примеры:
  3 12      → 3 подхода по 12 повторений
  4 10 50   → 4 подхода по 10 повторений с весом 50 кг
```

### Обработка кнопки "🔙 Назад"

Кнопка **🔙 Назад** всегда возвращает на **предыдущий экран**:

```
EXERCISE_DETAIL ──► EXERCISE_SELECT ──► CATEGORY_SELECT ──► MAIN_MENU
```

При возврате:
- Текущая тренировка **НЕ отменяется**
- Выбранные упражнения **сохраняются**
- Можно продолжить добавлять упражнения

## Развертывание на Railway

### Шаг 1: Подготовка репозитория

Убедитесь, что в репозитории есть все файлы:

**requirements.txt:**
```
python-telegram-bot==21.0
python-dotenv==1.0.0
flask==3.0.0
pytest==8.0.0
```

**runtime.txt:**
```
python-3.12.0
```

**Dockerfile:**
```dockerfile
FROM python:3.12-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["python", "bot.py"]
```

**Procfile:**
```
worker: python bot.py
```

### Шаг 2: Создание бота в Telegram

1. Откройте @BotFather в Telegram
2. Отправьте `/newbot`
3. Введите имя бота (например: "Fitness Tracker")
4. Введите username бота (например: "my_fitness_tracker_bot")
5. Скопируйте полученный токен

### Шаг 3: Деплой на Railway

1. Зайдите на https://railway.app
2. Нажмите "New Project"
3. Выберите "Deploy from GitHub repo"
4. Выберите ваш репозиторий
5. Railway автоматически определит Dockerfile

### Шаг 4: Настройка переменных окружения

В Railway Dashboard → Variables добавьте:

| Переменная | Значение |
|------------|----------|
| `TELEGRAM_BOT_TOKEN` | Токен от @BotFather |

Опционально:

| Переменная | Значение | По умолчанию |
|------------|----------|--------------|
| `DATABASE_PATH` | Путь к БД | `workout.db` |
| `LOG_LEVEL` | Уровень логирования | `INFO` |

### Шаг 5: Запуск

Railway автоматически запустит бота. Проверьте логи на наличие:

```
✅ Database initialized successfully
✅ Default data initialized
✅ Bot started successfully
```

## Решение проблем

### Проблема: RuntimeError: There is no current event loop

**Причина:** Python 3.14+ несовместим с python-telegram-bot 21.0

**Решение:** Используйте Dockerfile с Python 3.12-slim

### Проблема: ValueError: TELEGRAM_BOT_TOKEN not found

**Решение:** Добавьте переменную `TELEGRAM_BOT_TOKEN` в Railway Variables

### Проблема: База данных сбрасывается при перезапуске

**Причина:** Railway использует ephemeral filesystem

**Решения:**
1. Используйте Railway Volume (платно)
2. Подключите PostgreSQL addon в Railway
3. Используйте Turso/LibSQL (SQLite в облаке)

### Проблема: PTBUserWarning about per_message

**Причина:** Warning о настройках ConversationHandler

**Решение:** Добавьте `per_message=False` в ConversationHandler (не критично)

## Локальный запуск

```bash
# Клонировать репозиторий
git clone <repo-url>
cd tranie_bot

# Создать виртуальное окружение
python3 -m venv .venv
source .venv/bin/activate  # Linux/Mac
# .venv\Scripts\activate   # Windows

# Установить зависимости
pip install -r requirements.txt

# Создать .env файл
cp .env.example .env
# Отредактировать .env, добавить токен

# Запустить
python bot.py
```
